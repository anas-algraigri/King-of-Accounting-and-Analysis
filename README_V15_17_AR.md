# تحديث v15.17 - حساب مستقل مستوى سادس لكل عميل/مورد وربط فواتير الآجل

تم تنفيذ التحديث على مراحل داخل `index.html` بدون تغيير آلية القيود أو الفواتير أو الرواتب أو المرفقات أو الربط السحابي.

## ما تم تطبيقه
- أي عميل جديد يتم فتحه من شاشة العملاء/الموردين أو من الفواتير ينشأ له حساب مستقل تحت بند العملاء كمستوى سادس.
- أي مورد جديد يتم فتحه من شاشة العملاء/الموردين أو من الفواتير ينشأ له حساب مستقل تحت بند الموردين كمستوى سادس.
- تم إلغاء إنشاء حسابات الأطراف كمستوى سابع، مع ترحيل حسابات v15.16 القديمة تلقائيًا إلى مستوى سادس عند تشغيل الربط الشامل أو عند فتح النظام.
- فواتير الآجل أصبحت تستخدم حساب العميل أو المورد المرتبط بدل الحساب العام للعملاء أو الموردين.
- الحساب يظهر مباشرة لأنه أصبح حساب ترحيل نشط مستوى سادس في:
  - دليل الحسابات
  - شجرة الحسابات
  - بحث الحساب في القيود
  - الفواتير
  - السندات
  - التقارير
  - كشف الحساب ودفتر الأستاذ
- عند تعديل اسم أو رقم حساب عميل/مورد من دليل الحسابات أو الشجرة يتم تحديث الربط في سجل العملاء/الموردين والفواتير والقيود والتقارير.
- عند تعديل اسم العميل/المورد من سجل العملاء/الموردين أو من الفواتير يتم تحديث اسم حسابه المرتبط في الدليل والشجرة وباقي العمليات.
- تم تحسين مؤشر الترابط لإظهار أي عميل/مورد لا يملك حساب مستوى سادس صحيح.

## ملاحظات مهمة
- لم يتم تغيير طريقة إنشاء القيود أو احتساب الفواتير أو الرواتب أو المرفقات أو Supabase.
- عند رفع النسخة، استخدم زر **تحديث وربط شامل مالي وغير مالي** مرة واحدة لترحيل أي بيانات قديمة موجودة في المتصفح أو السحابة.

## طريقة الرفع على Vercel
ارفع ملف `index.html` بدل الملف الحالي في مشروعك، ثم اعمل Commit changes وانتظر تحديث Vercel.
