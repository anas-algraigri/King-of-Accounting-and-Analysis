# تحديث v15.17 - ربط العملاء والموردين بحسابات مستوى سادس

تم تطبيق التحديث على مراحل بدون تغيير آلية القيود أو الفواتير أو الرواتب أو المرفقات أو الربط السحابي.

## المرحلة 1: إنشاء حساب العميل/المورد
- أي عميل جديد يتم إنشاؤه من صفحة العملاء/الموردين أو من الفواتير ينشأ له حساب مستقل تحت بند العملاء كمستوى سادس.
- أي مورد جديد يتم إنشاؤه من صفحة العملاء/الموردين أو من الفواتير ينشأ له حساب مستقل تحت بند الموردين كمستوى سادس.
- تم منع إنشاء حسابات عملاء/موردين كمستوى سابع، مع ترحيل الحسابات القديمة المرتبطة إلى مستوى سادس قدر الإمكان.

## المرحلة 2: الظهور والبحث
- حساب العميل/المورد يظهر في دليل الحسابات وشجرة الحسابات.
- يظهر في بحث الحساب داخل القيود، الفواتير، السندات، كشف الحساب، دفتر الأستاذ، ميزان المراجعة، والتقارير.

## المرحلة 3: فواتير الآجل
- فواتير المبيعات الآجلة تستخدم حساب العميل المرتبط بدلاً من حساب العملاء العام.
- فواتير المشتريات الآجلة تستخدم حساب المورد المرتبط بدلاً من حساب الموردين العام.
- الفواتير النقدية والبنكية وجاري المالك بقيت بنفس آليتها السابقة.

## المرحلة 4: التحديث المتبادل
- عند تعديل اسم أو رقم حساب العميل/المورد من الدليل أو الشجرة يتم تحديث العميل/المورد والفواتير والقيود المرتبطة.
- عند تعديل العميل/المورد من سجله يتم تحديث حسابه المرتبط والفواتير والقيود المرتبطة.

## ملاحظات
- لم يتم تغيير آلية القيود أو الفواتير أو الرواتب أو المرفقات أو الربط السحابي.
- تم اختبار التحقق البرمجي للملف وتشغيل فحوصات منطقية لإنشاء عميل ومورد وفواتير آجلة وربطها بالحسابات.
